# Правила астрологічних розрахунків

Цей документ описує поточну логіку ядра Astroprocessor для управителів, зв'язків полів / домів за підходом СПбАА та відображення таблиці у стилі Sotis.

## Джерела і базові припущення

- У Sotis Online є окремий інструмент "Связи полей гороскопа (СПбАА)": https://sotis-online.ru/
- Для підходу СПбАА базовою системою домів є Кох. У Astroprocessor Кох стоїть за замовчуванням, але користувач може вручну обрати Плацидус у селекті.
- У матеріалах Шестопалова елемент дому визначається як планета, що стоїть у домі, або планета, що ним управляє. Зв'язок може виникати через аспект або через положення / управління.
- У таблиці Sotis-подібного типу клітинка читається як кількість плюсових і мінусових зв'язків між двома домами.

Додаткові відкриті описи методики:

- https://astrolog4you.com/articles/post/svazi-domov-otlicie-dvuh-podhodov
- https://astrolog4you.com/articles/post/svazi-domov-sintez-vseh-pokazatelej-goroskopa
- https://studfile.net/preview/4271974/

## Система домів

Для професійного режиму СПбАА використовуємо Кох:

```text
houseSystem = "koch"
```

Якщо користувач у формі обрав іншу систему, зв'язки будуються на фактично розрахованих домах цієї системи. Для порівняння із Sotis / СПбАА треба залишати Кох.

## Управителі знаків

Управитель знака залежить від того, чи є планета ретроградною. Меркурій і Венера завжди управляють двома знаками. Нижчі в таблиці планети у директному русі управляють одним знаком, у ретроградному - двома.

| Планета | Директний рух | Ретроградний рух |
|---|---|---|
| Сонце | Лев | Лев |
| Місяць | Рак | Рак |
| Меркурій | Близнюки, Діва | Близнюки, Діва |
| Венера | Телець, Терези | Телець, Терези |
| Марс | Скорпіон | Скорпіон, Овен |
| Юпітер | Стрілець | Риби, Стрілець |
| Сатурн | Козеріг | Козеріг, Водолій |
| Уран | Водолій | Водолій, Козеріг |
| Нептун | Риби | Риби, Стрілець |
| Плутон | Овен | Овен, Скорпіон |

Ліліт не управляє жодним знаком. Вона завжди додається як фіксований управитель 8 дому:

```text
Lilith -> 8 дім
```

## Як визначається управитель дому

Для кожного дому збираються всі активні управителі:

1. Береться знак на куспіді дому.
2. Для цього знака береться активний управитель з урахуванням руху планети.
3. Якщо другий або третій знак усередині дому займає більше ніж 12.5 градусів, управитель цього знака також додається як співуправитель.
4. Ліліт окремо додається до 8 дому як fixed-house ruler незалежно від знака на куспіді.

Якщо той самий управитель потрапляє в один дім кількома шляхами, для зв'язків він враховується як елемент цього дому один раз на роль.

## Елемент дому

Планета відповідає за дім у двох випадках:

```text
planet P відповідає за house H, якщо:
1. P стоїть у H;
2. P управляє H.
```

Це утворює набір відповідальностей планети:

```text
R(P) = placementHouses(P) + ruledHouses(P)
```

Наприклад:

```text
Сонце стоїть у 2 домі і управляє 12 домом
R(Сонце) = [2 placement, 12 ruler]
```

Хірон не використовується у зв'язках будинків. Він може відображатися на карті та брати участь в інших розділах аналізу, але для таблиці зв'язків полів не створює відповідальностей ні через положення, ні через аспекти.

## Тони зв'язків

Аспекти переводяться у тон зв'язку так:

| Аспект | Тон у таблиці |
|---|---|
| Трин | + гармонійний |
| Секстиль | + гармонійний |
| Квадрат | - напружений |
| Опозиція | - напружений |
| З'єднання | + або - за правилом злих домів |

Для з'єднання, положення і управління знак зв'язку визначається участю злих домів:

```text
злі доми = 6, 8, 12
```

Якщо в парі домів є хоча б один злий дім, зв'язок мінусовий. Якщо злих домів у парі немає, зв'язок плюсовий.

```text
2 + 11 = плюс
2 + 12 = мінус
11 + 12 = мінус
6 + 6 = мінус
```

## Формула зв'язків домів

Зв'язки будуються у два проходи.

### 1. Внутрішні зв'язки планети

Для кожної планети беремо всі її відповідальності `R(P)` і створюємо пари між кожними двома відповідальностями.

```text
для P:
  для кожної пари h1, h2 з R(P):
    якщо h1 або h2 належить до 6/8/12:
      додати мінусовий зв'язок h1 - h2
    інакше:
      додати плюсовий зв'язок h1 + h2
```

Якщо планета стоїть у тому самому домі, яким управляє, дозволена діагональна формула:

```text
6 + 6
8 + 8
```

### 2. Аспектні зв'язки між планетами

Для аспекту між планетами `A` і `B` беремо всі відповідальності обох планет:

```text
R(A) x R(B)
```

Після цього створюємо всі унікальні пари домів між відповідальностями першої та другої планети. Внутрішні зв'язки кожної планети вже враховані в першому проході "положення / управління" і не дублюються аспектом.

Тон аспектної пари:

```text
трин / секстиль = плюс
квадрат / опозиція = мінус
з'єднання = плюс, якщо в парі немає 6/8/12; мінус, якщо 6/8/12 залучений
```

Приклад:

```text
Сонце стоїть у 2 домі і управляє 12 домом.
Сатурн стоїть у 6 домі і управляє 5 домом.
Сонце має трин із Сатурном.
```

Отримуємо гармонійні зв'язки:

```text
2 + 6
2 + 5
12 + 6
12 + 5
```

Окремо через положення / управління цих планет також будуть враховані внутрішні зв'язки:

```text
Сонце: 2 + 12
Сатурн: 6 + 5
```

Якщо аспект або внутрішня відповідальність дає пару одного дому з самим собою, вона не відкидається:

```text
6 + 6
8 + 8
12 + 12
```

## Формат клітинки таблиці

Кожна клітинка таблиці показує тільки плюсові та мінусові зв'язки:

```text
+N  кількість плюсових зв'язків
-K  кількість мінусових зв'язків
```

Приклад:

```text
+3
-1
```

Це означає: 3 плюсові та 1 мінусовий зв'язок. Загальна кількість у звичайних клітинках не показується.

Таблиця симетрична: зв'язок `4 + 7` дорівнює `7 + 4`. Діагональ використовується для формул типу `6 + 6`.

При наведенні на клітинку показується список конкретних зв'язків:

```text
+ 2 Сонце (П) - 4 Юпітер (У) 0
- 3 Місяць (П) - 6 Меркурій (У) 90
```

Позначення:

```text
П = положення планети в домі
У = управління домом
0 / 60 / 90 / 120 / 180 = точний кут аспекту
```

## Підсумковий рядок

Рядок `Σ` підсумовує всі зв'язки, у яких бере участь кожен дім:

```text
Σ(H) = всі зв'язки H з іншими домами + діагональні зв'язки H + H
```

Якщо зв'язок `2 + 8` має `+1`, він додається і до підсумку 2 дому, і до підсумку 8 дому. Якщо зв'язок `8 + 8`, він додається до підсумку 8 дому один раз.

У підсумковому рядку для кожного дому показуються:

```text
+N  плюсові зв'язки дому
-K  мінусові зв'язки дому
=T  усього зв'язків дому
```

## Правило трьох указань

Для подієвої формули орієнтиром є 3 або більше зв'язки потрібних домів.

```text
позитивна формула: 3+ плюсових указань
негативна формула: 3+ мінусових указань
```

Нейтральні зв'язки в таблиці більше не виводяться. З'єднання і зв'язки положення / управління переводяться в плюс або мінус через правило злих домів 6/8/12.

## Обмеження поточної реалізації

- Зв'язки домів не рахуються без точного часу народження, бо тоді немає надійних домів.
- Для повного порівняння із СПбАА треба використовувати систему Кох.
- У поточній версії враховані положення планети в домі, управління домами, співуправителі за знаком більше 12.5 градусів, фіксоване управління Ліліт 8 домом і мажорні аспекти. Хірон у зв'язках будинків не враховується.
- Взаємні рецепції та окремий список сигніфікаторів для всіх домів поки не винесені в окрему налаштовувану таблицю.
